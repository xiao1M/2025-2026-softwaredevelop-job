### 实验二-任务三：完成面向AI应用的Compose布局
根据实验要求，设计如下 AI 图像识别应用界面：

## 一、任务目标

设计并实现一个面向 AI 图像识别应用的 Android Compose 界面，该界面能够：

- 展示相机预览区域
- 显示识别结果信息
- 提供用户操作按钮
- 实现界面状态管理

## 二、界面布局设计

### 2.1 整体布局结构

采用 **Column** 垂直布局组织整个页面，从上到下依次排列四个主要区域：

| 区域 | 布局组件 | 功能描述 |
|------|----------|----------|
| 顶部栏 | Row | 显示应用标题和菜单入口 |
| 预览区 | Box + Card | 用于后续集成 CameraX 相机预览，当前使用占位符 |
| 结果区 | Card + Column | 展示 AI 模型的识别结果信息 |
| 按钮区 | Row + Column | 提供四个操作按钮 |

### 2.2 各区域详细说明

#### （1）顶部栏

- 左侧：应用标题文字"AI 图像识别"
- 右侧：菜单按钮（预留设置入口）
- 使用 `Row` 实现水平排列，`Arrangement.SpaceBetween` 实现两端对齐

#### （2）预览区

- 使用 `Card` 组件包裹，增加圆角和阴影效果
- 内部使用 `Box` 作为占位容器，居中显示图标和提示文字
- 根据是否有识别活动，显示相机图标或图片占位图标
- 高度固定为 280dp

#### （3）结果区

- 使用 `Card` 卡片形式展示识别结果
- 卡片内使用 `Column` 垂直排列信息行
- 显示内容：
  - 模型名称（如 MobileNetV3、ResNet50）
  - 识别结果（如识别出的物体名称）
  - 置信度（百分比）
  - 推理时间（毫秒）
- 识别结果使用更大的字体和主题色突出显示

#### （4）按钮区

- 第一行：拍照识别、相册导入
- 第二行：切换模型、清空结果
- 每行使用 `Row` 水平排列，按钮之间间距 12dp
- 两行之间间距 12dp
- 使用 `Column` 组织两行按钮
- 按钮高度统一为 56dp，圆角 8dp

## 三、状态管理

使用 Compose 的状态管理机制来响应用户交互：

| 状态变量 | 类型 | 初始值 | 用途 |
|----------|------|--------|------|
| modelName | String | "MobileNetV3" | 存储当前使用的模型名称 |
| recognitionResult | String | "待识别" | 存储识别结果 |
| confidence | String | "0.00" | 存储置信度 |
| inferenceTime | String | "0" | 存储推理耗时 |
| isPreviewActive | Boolean | false | 控制预览区显示状态 |

所有状态使用 `remember { mutableStateOf() }` 包装，确保 UI 在状态变化时自动重组更新。

## 四、交互逻辑说明

### 4.1 拍照识别按钮

**触发动作：**

- 将 `isPreviewActive` 设为 `true`，预览区切换为相机图标
- 将识别结果显示为"猫"
- 将置信度设为"95.00%"
- 将推理时间设为"124ms"

**代码逻辑：**

```kotlin
onCameraClick = { 
    isPreviewActive = true
    recognitionResult = "猫"
    confidence = "95.00"
    inferenceTime = "124"
}
```
